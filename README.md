# texture5 - テクスチャ:第９回 視点マッピング サンプルプログラム

## 1. 概要

このプログラムは、OpenGL における「テクスチャマッピング (Texture Mapping)」の基礎を学ぶための、学生向けのサンプルプログラムです。本プログラムは、以下のブログ記事の解説に沿って学習を進めるための雛形として提供されています。

- [テクスチャ:第９回 視点マッピング](https://tokoik.github.io/blog/opengl/%E3%83%86%E3%82%AF%E3%82%B9%E3%83%81%E3%83%A3/2004/12/30/texture.html)

今回はテクスチャマッピングにおいて、**常に視点から見た方向からテクスチャをマッピングする**方法について学びます。

テクスチャ座標を自動生成を用いて、ヘッドライトのように、図形に対してテクスチャを投影するようにマッピングします。

プログラムを実行すると、表面にタイヤのパターン画像（tire.raw）が貼り付けられた２つの箱が表示され、そのテクスチャが自動で回転します。マウスのドラッグで図形を回転しても、貼り付けられているテクスチャの投影方向は変化しません。

![視点マッピング](https://tokoik.github.io/blog/assets/images/texture5.gif)

## 2. ビルド方法

このプログラムは CMake を用いてビルドを構成しています。各プラットフォームごとの手順は以下の通りです。なお、プログラムをビルドするためのバイナリディレクトリは、バージョン管理ファイル（.gitignore）の設定に合わせて build という名前にします。

### 2.1 Windows (Visual Studio 2022 の場合)

1. コマンドプロンプトまたは PowerShell を開き、このプロジェクトのディレクトリに移動します。

2. 以下のコマンドを実行してビルドディレクトリを作成し、CMake で構成を行います。

```bat
mkdir build
cd build
cmake .. -G "Visual Studio 17 2022"
```

3. 生成された build フォルダ内の texture5.sln を Visual Studio で開きます。

4. ソリューションエクスプローラーで texture5 プロジェクトを右クリックし、「スタートアップ プロジェクトに設定」を選択します。

5. 「ローカル Windows デバッガー」をクリックするか、F5 キーを押してビルドおよび実行します。

### 2.2 macOS (Xcode の場合)

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。

2. 以下のコマンドを実行してビルドディレクトリを作成し、Xcode 用のプロジェクトを生成します。

```sh
mkdir build
cd build
cmake .. -G Xcode
```

3. 生成された build/texture5.xcodeproj を Xcode で開きます。

4. 左上のスキーム選択（再生ボタンの横）が texture5 になっていることを確認します。

5. 「Run」ボタン（再生ボタン）をクリックするか、Command + R を押してビルドおよび実行します。

### 2.3 Ubuntu Linux

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。

2. 必要なパッケージ（freeglut3-dev や pkg-config など）がインストールされていることを確認し、以下のコマンドでビルドします。

```sh
mkdir build
cd build
cmake ..
make
```

## 3. 使い方

### 3.1 プログラムの起動方法

各OSとも、ビルド後に生成されるバイナリディレクトリ (build) やそのサブフォルダから起動します。（※ CMake の設定により、Windows や Xcode では Debug などのフォルダ下に実行ファイルが置かれることがあります）

- **Windows**

Visual Studio 上で「ローカル Windows デバッガー」をクリックして実行するか、またはコマンドプロンプトから以下のコマンドで起動します。

```cmd
cd build\Debug
texture5.exe
```

- **macOS**

Xcode 上で左上の「Run（再生ボタン）」をクリックするのが楽です。これにより texture5.app アプリケーションバンドルとして自動的に実行されます。アプリケーションバンドルを直接起動するなら、Finder から build/Debug/texture5.app をダブルクリックするか、ターミナルから open build/Debug/texture5.app を実行します (この場合はエラーメッセージ等が表示されません)。

- **Ubuntu Linux**

ターミナルから以下のコマンドで実行ファイル（バイナリ）を直接起動します。

```sh
cd build
./texture5
```

### 3.2 操作方法

- **マウスの左ボタンでドラッグ**: 表示されている**オブジェクト（箱）**を３次元的に回転させることができます。テクスチャ画像はオブジェクトとは独立してスクリーン上に固定して張り付いたままになります。

- **キーボードの q, Q または ESC キー**: プログラムを終了します。

- **アニメーション**: テクスチャ画像がスクリーン（ウィンドウ）上で自動的に回転し続けます。

## 4. 解説

このプログラムでは、テクスチャ座標の自動生成（`GL_EYE_LINEAR`）と、テクスチャ行列における透視変換行列・モデルビュー変換行列の乗算を組み合わせることで、**スクリーン（ウィンドウ）の表示領域全体にテクスチャを貼り付けるようなマッピング**を実現しています。

### 4.1 視点の回転（オブジェクトの回転） (`display()` 関数)

`texture4` ではテクスチャ行列に適用してテクスチャの投影方向を回転させていたトラックボールの処理を、`texture5` ではモデルビュー変換行列に適用するように変更しています。これにより、マウスドラッグでオブジェクト自身が回転するようになっています。

```c
  /* トラックボール処理で図形を回転 */
  glMultMatrixd(trackballRotation());
```

### 4.2 テクスチャ生成関数の設定 (`scene()` 関数)

`GL_EYE_LINEAR` によるテクスチャ座標生成のパラメータを設定します。
`glTexGendv` を呼び出した時点でのモデルビュー変換行列の逆行列が `genfunc` に掛けられるため、生成されるテクスチャ座標は「オブジェクト空間の頂点座標」と一致するようになります。今回は、シーンを描画する直前の `scene()` 関数内でこれを行っています。

```c
  /* テクスチャ生成関数の設定 */
  glTexGendv(GL_S, GL_EYE_PLANE, genfunc[0]);
  // ... (T, R, Q も同様)
```

### 4.3 透視変換・モデルビュー変換行列の取得とテクスチャ行列への適用 (`display()` 関数)

テクスチャ行列で「クリップ座標」へ変換するために、現在のモデルビュー変換行列と透視変換行列を `glGetDoublev` で取り出します。その後、テクスチャ行列に対して `proj` と `model` を掛けることで、オブジェクト空間の頂点座標に対してモデルビュー変換と透視変換が適用され、クリップ座標系の値が得られます。

さらに `glScaled(0.5, 0.5, 1.0)` と `glTranslated(0.5, 0.5, 0.0)` を適用することで、クリップ座標系（-1.0〜1.0）をテクスチャ座標系（0.0〜1.0）の範囲にマッピングしています。`glRotated(t * 360.0, 0.0, 0.0, 1.0)` は時間 `t` に伴ってテクスチャが回転するアニメーションを加える処理です。

```c
  /* テクスチャ行列の設定 */
  glMatrixMode(GL_TEXTURE);
  glLoadIdentity();
  glTranslated(0.5, 0.5, 0.0);
  glRotated(t * 360.0, 0.0, 0.0, 1.0);
  glScaled(0.5, 0.5, 1.0);

  /* 現在の透視変換行列とモデルビュー変換行列を取り出す */
  GLdouble model[16], proj[16];
  glGetDoublev(GL_MODELVIEW_MATRIX, model);
  glGetDoublev(GL_PROJECTION_MATRIX, proj);

  /* テクスチャ行列に取り出した透視変換行列とモデルビュー変換行列を掛ける */
  glMultMatrixd(proj);
  glMultMatrixd(model);
```

これにより、画面の表示領域全体に1枚のテクスチャが固定されて張り付いたような視覚効果が得られ、オブジェクトを回転させてもテクスチャ自体はスクリーンに対して動かず、オブジェクトがそのテクスチャを切り抜く「窓」のような役割を果たします。
